## Key Insights

- Need `articles` relation on User model
- Slug must be unique per locale (composite unique constraint)

## Requirements

### Functional
- Article model: title, slug, description, content (HTML), locale, status, categories (string[]), tags (string[]), author relation
- ArticleStatus enum: DRAFT, PENDING, PUBLISHED, REJECTED
- Admin review fields: reviewNote, reviewedAt, reviewedBy
- Gradient fields for card display consistency: gradientFrom, gradientTo
- Timestamps: createdAt, updatedAt, publishedAt

### Non-Functional
- Indexes on: status, authorId, locale, slug
- Composite unique on [slug, locale]
- Cascade delete when user deleted

## Architecture

```
ArticleStatus enum: DRAFT | PENDING | PUBLISHED | REJECTED

Article {
  id           String (cuid)
  title        String
  slug         String
  description  String
  content      String (HTML body)
  locale       String (default: "en")
  status       ArticleStatus (default: DRAFT)
  categories   String[]
  tags         String[]
  gradientFrom String?
  gradientTo   String?
  coverImage   String?
  reviewNote   String?
  reviewedAt   DateTime?
  reviewedBy   String?
  publishedAt  DateTime?
  authorId     String -> User.id (CASCADE)
  createdAt    DateTime
  updatedAt    DateTime
}

User.articles -> Article[] (one-to-many)
```

## Implementation Steps

1. Add `ArticleStatus` enum to `prisma/schema.prisma`:
   ```prisma
   enum ArticleStatus {
     DRAFT
     PENDING
     PUBLISHED
     REJECTED
   }
   ```

2. Add `Article` model to `prisma/schema.prisma`:
   ```prisma
   model Article {
     id           String        @id @default(cuid())
     title        String
     slug         String
     description  String        @default("")
     content      String        @default("")
     locale       String        @default("en")
     status       ArticleStatus @default(DRAFT)
     categories   String[]
     tags         String[]
     gradientFrom String?
     gradientTo   String?
     coverImage   String?
     reviewNote   String?
     reviewedAt   DateTime?
     reviewedBy   String?
     publishedAt  DateTime?
     authorId     String
     author       User          @relation(fields: [authorId], references: [id], onDelete: Cascade)
     createdAt    DateTime      @default(now())
     updatedAt    DateTime      @updatedAt

     @@unique([slug, locale])
     @@index([status])
     @@index([authorId])
     @@index([locale])
   }
   ```

3. Add `articles` relation to `User` model:
   ```prisma
   articles  Article[]
   ```

4. Run migration:
   ```bash
   npx prisma migrate dev --name add-article-model
   ```

5. Verify generated client types include Article + ArticleStatus
