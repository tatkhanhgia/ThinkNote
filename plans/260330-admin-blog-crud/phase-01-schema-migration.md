## Overview
- **Priority:** P1 (blocks all other phases)
- **Status:** Complete
- **Effort:** 1.5h

Extend Article model with ContentType enum, mood, and readingTime fields. Migrate existing file-based blog posts to DB.

## Key Insights
- Article model already has: title, slug, description, content, locale, status, categories, tags, coverImage, gradientFrom, gradientTo
- Blog needs additionally: mood (string), readingTime (int), type (enum)
- Existing blog posts: 2 files (en + vi `welcome-to-my-blog.md`)
- `@@unique([slug, locale])` already exists — blog posts will use same constraint
- Blog admin workflow: DRAFT ↔ PUBLISHED only (no PENDING/REJECTED)

## Requirements

### Functional
- Add ContentType enum (ARTICLE, BLOG_POST)
- Add `type`, `mood`, `readingTime` fields to Article
- Index on `type` for efficient filtering
- Migrate 2 existing blog posts from files to DB

### Non-functional
- Zero downtime migration (additive changes only)
- Existing article API must not break
- Default `type = ARTICLE` for backwards compatibility

## Architecture

```
Article model (extended)
├── type: ContentType (ARTICLE | BLOG_POST)  ← NEW
├── mood: String?                             ← NEW (blog only)
├── readingTime: Int?                         ← NEW (auto-calculated)
├── ... (all existing fields unchanged)
└── @@index([type])                           ← NEW
```

## Implementation Steps

1. **Add ContentType enum to schema.prisma**
   ```prisma
   enum ContentType {
     ARTICLE
     BLOG_POST
   }
   ```

2. **Add fields to Article model**
   ```prisma
   type        ContentType @default(ARTICLE)
   mood        String?
   readingTime Int?
   ```

3. **Add index**
   ```prisma
   @@index([type])
   ```

4. **Run migration**
   ```bash
   npm run db:push
   npm run db:generate
   ```

5. **Create migration script** to import existing blog posts from `src/data/blog/{locale}/*.md` into DB
   - Read each .md file with gray-matter
   - Calculate readingTime
   - Create Article with type=BLOG_POST, status=PUBLISHED
   - Use admin user as author

6. **Verify** existing article queries still work (type defaults to ARTICLE)
